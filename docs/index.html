Forktex Documentation

Your development companion — from ideation to production. Forktex combines an AI agent framework, cloud infrastructure management, and delivery standard enforcement into a single forktex command.

Three Pillars

Intelligence

AI-powered agent with tool calling. Reads code, runs commands, applies patches.

forktex chat forktex ask forktex run

Cloud

Deploy and manage infrastructure. Blue-green deploys from forktex.json manifests.

forktex cloud up forktex cloud deploy

FSD

Delivery standard. Verify compliance, generate ISO audit evidence.

forktex fsd check forktex fsd report

Installation

pip install forktex

# Optional: web scraping support (Playwright)
pip install forktex[web]
playwright install

Intelligence (AI Agent)

Tool-calling LLM loop with full project awareness, powered by the Forktex Intelligence API.

forktex                              # Interactive chat
forktex ask "What does main.py do?"  # Single question
forktex run "Fix the auth bug"       # Full orchestration

Slash commands in interactive mode: /status /tools /model /clear /help

Cloud (Infrastructure)

Deploy from forktex.json manifests. Blue-green deployments, DNS, SSL, vault secrets.

forktex cloud connect                  # Connect to controller
forktex cloud up --env local --build   # Start local stack
forktex cloud deploy <server-id>     # Deploy to production
forktex cloud logs <server-id>       # View logs
forktex cloud server list            # List servers

FSD (Delivery Standard)

Verify and enforce the ForkTex Standard for Delivery. Atoms, domains, levels, and ISO traceability live behind one portable standard surface.

forktex fsd check                    # Check compliance
forktex fsd check --level L2         # Require L2 (CI gate)
forktex fsd report                   # Run gates, generate evidence

Outputs both JSON (machine-readable) and HTML (business/audit) to .forktex/fsd/evidence/.

Git (Multi-repo)

Operate across multiple repositories at once.

forktex git status-all                         # All sibling repos
forktex git branch-all forktex-standard        # Branch all repos
forktex git checkout-all master                # Switch all to master
forktex git status-all -p docs -p cloud -p net # Specific repos

Scrape (Web)

Persistent Playwright browser + AI agent. Navigates, extracts data, remembers selectors via "truths".

forktex scrape "https://example.com" --goal "Extract products"

.forktex/ Directory

All forktex-managed project state lives here:

.forktex/
  config.json                # Project configuration
  cloud.json                 # Cloud context (project, server, env)
  intelligence.json          # Intelligence API settings
  docker-compose.local.yml     # Generated dev compose
  fsd/evidence/              # ISO audit evidence (gitignored)
  architecture/              # C4 architecture snapshots
  agents/history/            # Agent session state
  vault/{env}/secrets.enc    # Encrypted secrets
  scraper/truths/            # Per-domain selectors

Global config: ~/.forktex/config.toml, ~/.forktex/intelligence.json, ~/.forktex/cloud.json

Architecture

forktex                  pip install forktex (CLI + agents + FSD — this repo)
forktex-intelligence     pip install forktex-intelligence (AI SDK — own repo)
forktex-cloud            pip install forktex-cloud (Cloud SDK — own repo)
forktex-core             pip install forktex-core (PostgreSQL/Redis primitives — own repo)
src/forktex/
  agent/
    cli.py                 # Main CLI entry point
    fsd/                   # FSD command surface (check, report, makefile)
    cloud/                 # Cloud CLI commands + config persistence
    intelligence/          # Intelligence CLI + config persistence
    commands/git_cli.py    # Multi-repo git CLI
    tools/                 # 25+ agent tools
    scraper/               # Agentic web scraping
  intelligence/            # SDK re-export shim
  cloud/                   # SDK re-export shim
  core/                    # Paths, state, utilities
  fsd/                     # FSD evaluation engine
  models/                  # Shared data models

Local Setup

# Editable install (forktex-py only — pulls SDKs from PyPI)
cd forktex-py
python -m venv .venv && source .venv/bin/activate
pip install -e .[dev]

# Full ecosystem dev (one-time, sibling-layout):
pip install -e ../intelligence/sdk \
            -e ../cloud/sdk \
            -e ../core-py \
            -e .[dev]
